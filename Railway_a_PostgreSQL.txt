Cambios a realizar para la inserción de datos de la nube (Railway) a la base de Docker

En el Docker se le debe de instalar el archivo Python llamado "migrar_a_python"

En caso de que falle ejecutar estos scripts en sql (pgAdmin) 

ALTER TABLE edge_data ADD COLUMN IF NOT EXISTS begin_time DOUBLE PRECISION;
ALTER TABLE edge_data ADD COLUMN IF NOT EXISTS end_time DOUBLE PRECISION;

ALTER TABLE simulation_stats ADD COLUMN IF NOT EXISTS step INTEGER;
ALTER TABLE simulation_stats ADD COLUMN IF NOT EXISTS vehicles_loaded INTEGER;
ALTER TABLE simulation_stats ADD COLUMN IF NOT EXISTS vehicles_inserted INTEGER;
ALTER TABLE simulation_stats ADD COLUMN IF NOT EXISTS vehicles_running INTEGER;
ALTER TABLE simulation_stats ADD COLUMN IF NOT EXISTS vehicles_waiting INTEGER;

ALTER TABLE pt_stops ADD COLUMN IF NOT EXISTS delay DOUBLE PRECISION;
ALTER TABLE pt_stops ADD COLUMN IF NOT EXISTS passengers_loaded INTEGER;

ALTER TABLE vehicle_types ADD COLUMN IF NOT EXISTS description TEXT;

Y luego volver ejecutar script 

*en caso de que suceda un error pero hayan datos ya descargados ejecutar este script*

Script para borrar datos (solo datos)
TRUNCATE TABLE 
    pt_stops, 
    simulation_stats, 
    edge_data, 
    vehicle_trips, 
    simulation_runs,
    time_periods,
    vehicle_types
RESTART IDENTITY CASCADE;

Script para reingresar los datos estáticos 
INSERT INTO time_periods (period_name, start_hour, end_hour, description) VALUES
    ('MADRUGADA', 0, 5, 'Madrugada (00:00-05:00)'),
    ('MAÑANA', 5, 7, 'Mañana temprano (05:00-07:00)'),
